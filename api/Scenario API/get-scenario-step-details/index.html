API Documentation

Fetches comprehensive information about test Scenarios within your project. This endpoint supports: - **Pagination**: Control response size with configurable page numbers and sizes (max 25 items per page) - **Flexible Filtering**: Filter by specific Scenario IDs or requirement traceability keys - **Optional Detailed Information**: Selectively include Scenario metadata such as step details, requirement traceability in the response - **Version Control**: Choose between base version or user's checked-out version of Scenarios

POST {accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectName}/scn/scenarios/{scenarioType}/fetch-details

Retrieve Scenario details with advanced filtering and pagination support

Fetches comprehensive information about test Scenarios within your project. This endpoint supports:

  • Pagination: Control response size with configurable page numbers and sizes (max 25 items per page)
  • Flexible Filtering: Filter by specific Scenario IDs or requirement traceability keys
  • Optional Detailed Information: Selectively include Scenario metadata such as step details, requirement traceability in the response
  • Version Control: Choose between base version or user's checked-out version of Scenarios

Request

Method: POST URL: {accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectName}/scn/scenarios/{scenarioType}/fetch-details

Headers

HeaderValue
user_idLogin username (email)
api_keyAPI key of the user for authentication
Content-Typeapplication/json

Parameters

ParameterTypeRequiredDescription
accelqLoginUrlstringYesACCELQ login URL (e.g., https://yourcompany.accelq.io)
tenantCodestringYesURL parameter: tenantCode
projectNamestringYesURL parameter: projectName
scenarioTypestringYesScenario Type
showMyViewbooleanNo
Controls scenario version selection:
  • false (default): Returns base version of Scenarios regardless of your checked out activity
  • true: Returns user's checked-out version if available, otherwise falls back to base version |

| user_id | string | Yes | Login username (email) |

| api_key | string | Yes | API key of the user for authentication. Obtain this from your AccelQ user profile. |

| pageNumber | integer | No | Page number for pagination. Use this to navigate through large result sets efficiently. |

| pageSize | integer | No | Number of Scenarios to return per page.

Constraints:
  • Minimum: 1
  • Maximum: 25
  • Default: 20 |
