GET Get all Template Test Jobs
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/templates
This API retrieves a comprehensive list of all template jobs that have been created and associated with a specific project.
GET Fetch test case status summary by users of manul test run
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/runs/manual/{runPid}/tc-summary-by-users
It fetches summary of all test cases status updated by each user for a specific test run. It helps tracking user-wise contributions and changes made during that run.
POST Create a new Recurring Test Job
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/recurring
Creates a new recurring test job for the specified project based on the provided job configuration. This API sets up a recurrence template for the job, with actual job runs being triggered automatically when the specified recurrence conditions are met.
POST Configure and Run a new Test Job instantly
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/run
Initiates a test job immediately with the specified project configuration. The job begins execution as soon as an agent becomes available.
POST Create a new Template Test Job
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/templates
Creates a new template test job for the specified project using the provided job configuration. This does not execute the job but serves as a template that stores the job configurations. Actual job runs can be triggered externally as needed, using the job ID of the template created by this API.
POST Instantiates a run from the template job
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/templates/{templateJobId}/run
This API Instantiates a new job run by applying the configuration associated with the provided template job ID.
POST Retrieve Scenario details with advanced filtering and pagination support
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectName}/scn/scenarios/{scenarioType}/fetch-details
Fetches comprehensive information about test Scenarios within your project. This endpoint supports:
  • Pagination: Control response size with configurable page numbers and sizes (max 25 items per page)
  • Flexible Filtering: Filter by specific Scenario IDs or requirement traceability keys
  • Optional Detailed Information: Selectively include Scenario metadata such as step details, requirement traceability in the response
  • Version Control: Choose between base version or user's checked-out version of Scenarios
PATCH Process multiple Scenario updates with granular control and configurations
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectName}/scn/scenarios/{scenarioType}/bulk-update
Provides efficient bulk updates of Scenario metadata through flexible, operation-based requests. This API allows different Scenarios to be updated with distinct criteria and configurations. Each object in the payload targets specific Scenario IDs and applies the defined metadata changes, providing granular control over batch modifications.

POST Create a new Project
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects
Create a new project in the Tenant. Project name must adhere to the following criteria:
    1.Must always start with a alphabetic character
    2.Maximum length of 256 characters
    3.Only letters, digits, spaces, and underscores allowed
POST Clone a Project
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects/{pjPid}/clone
Clone an existing project and create a new one with an optional branching.
PUT Update an existing Project
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects/{pjPid}
Update the details (name and description) of an existing project.
PATCH Configures Requirement Traceability integration for given Projects
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects/requirement-traceability/configure
This API configures Requirements Traceability Integration within the Project Configuration for a specified list of projects. It must be executed by a Tenant Admin to ensure access to all listed projects. Note: Any existing Requirements Traceability configuration will be overwritten. However, currently mapped tickets/stories from these integrations will be retained.
