GET List of App Environments
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/conf/app-environments
Returns a paginated list of application environments configured for the given Project.
POST Create App Environment
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/conf/app-environments
Creates a new application environment under the specified Project.
PUT Update App Environment
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/conf/app-environments/{appEnvironmentId}
Updates the name or description of an existing app environment.
DELETE Delete App Environment
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/conf/app-environments/{appEnvironmentId}
Delete an existing App Environment in the specified Project.
GET List Run Properties
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/res/run-property
Returns a paginated list of run properties configured for the current project.
GET Encrypt string.
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/res/encrypt
Encrypts the provided input string and returns the encrypted value.
POST Create Run Property
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/res/run-property
Creates a new run property for the specified project.
PUT Update Run Property
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/res/run-property/{runPropertyPid}
Updates the name of an existing run property identified by its PID.
GET List Job templates
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/templates
This API retrieves list of all template jobs that have been created and associated with a specific project.
GET Get manual run test case summary by users
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/runs/manual/{runPid}/tc-summary-by-users
Returns a user-wise summary of test case status updates for the specified manual test run. This helps identify which users updated which test cases during the run.
POST Create recurring test job
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/recurring
Creates a recurring test job template that automatically triggers executions based on the specified recurrence rules.
POST Run test job
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/run
Triggers an immediate execution of a Scenario or Test Suite using the provided job configuration. The job is queued and starts running as soon as a suitable agent becomes available.
POST Create Job template
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/templates
Creates a reusable job template that stores execution configuration for a Scenario or Test Suite. The executions can be triggered on demand using this job template Id
POST Trigger Run from Job template
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/test-exec/jobs/templates/{templateJobId}/run
Creates a new job run using the configuration stored in the specified job template. Optional run properties can be overridden for this execution.
POST Fetch Scenario details
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/scn/scenarios/{scenarioType}/fetch-details
Fetches comprehensive information about test Scenarios within your project. This endpoint supports:
  • Pagination: Control response size with configurable page numbers and sizes (max 25 items per page)
  • Flexible Filtering: Filter by specific Scenario IDs or requirement traceability keys
  • Optional Detailed Information: Selectively include Scenario metadata such as step details, requirement traceability in the response
  • Version Control: Choose between base version or user's checked-out version of Scenarios
PATCH Bulk update Scenario Requirement Traceability
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/{projectCode}/scn/scenarios/{scenarioType}/bulk-update
Updates requirement traceability for multiple Scenarios.
GET List projects
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects
Returns a paginated list of all projects available within the tenant.
POST Create project
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects
Create a new project in the Tenant. Project name must adhere to the following criteria:
    1.Must always start with a alphabetic character
    2.Maximum length of 256 characters
    3.Only letters, digits, spaces, and underscores allowed
POST Clone project
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects/{pjPid}/clone
Creates a new project by cloning configuration from an existing project.
PUT Update project
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects/{pjPid}
Updates the name and description of an existing project identified by its Pid.
PATCH Configure requirement traceability
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/projects/requirement-traceability/configure
This API configures Requirements Traceability Integration within the Project Configuration for a specified list of projects. It must be executed by a Tenant Admin to ensure access to all listed projects. Note: Any existing Requirements Traceability configuration will be overwritten. However, currently mapped tickets/stories from these integrations will be retained.
GET Get test job run status
{accelqLoginUrl}/awb/api/1.0/{tenantCode}/runs/{runPid}
Retrieves the current status and results of a test job using its run PID. This includes the overall job status, the total number of test cases, and the counts of passed, failed, and not-run cases. You can monitor the status of an ongoing job using the API described below
GET List subscriptions
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/subscriptions
Returns all subscriptions available in the tenant along with their associated license bundles and usage details.
GET Retrieve all users in the Tenant
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/users
Retrieve a list of all users in the tenant, including their assigned licenses and Project roles.

{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/users

{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/users/activate

PUT Deactivate User
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/users/deactivate/{userPid}
Deactivates the specified User
PUT Assign license to User
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/users/license
Assigns or updates license bundles for the specified user.
DELETE Delete User
{accelqLoginUrl}/awb/api/2.0/{tenantCode}/administration/users/{userPid}
Deletes the specified User from the tenant.
